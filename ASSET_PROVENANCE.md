# Asset provenance

Shyne Creator records the provenance of every bundled binary asset so releases can be audited without relying on memory.

## Bundled project assets

| Asset | Status | Notes |
|---|---|---|
| `assets/shyne_creator/icon.png` | Project asset; authorship record required before a public release | No external license metadata was found in the source snapshot. |
| `assets/shyne_creator/textures/gui/shyne_creator_logo.png` | Project asset; authorship record required before a public release | Currently byte-identical to the project icon. |
| Avatar, skill and power content | Not bundled | Shyne Creator is content-neutral; creators distribute these as separate packs. |

## Release rule

An asset may be published only when one of these is recorded:

1. created for Shyne, with its editable source or creation note;
2. licensed for redistribution, with author, source URL, license, and required attribution;
3. public domain, with a source citation.

Unknown assets must be replaced or excluded from release artifacts.
